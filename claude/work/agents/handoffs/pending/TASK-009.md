# TASK-009: フロントエンドの言語セクション表示対応

## タスク概要: GitHub Trendingの言語セクションヘッダーを適切に表示するためのフロントエンド修正

## 変更予定ファイル: 
- nook/frontend/src/components/ContentCard.tsx
- nook/frontend/src/App.tsx（必要に応じて）

## 前提タスク: TASK-007（APIが言語セクションヘッダーを返すようになっていること）

## worktree名: worktrees/TASK-009-frontend-section-display

## 作業内容:

1. **ContentCardコンポーネントの修正**
   - source="github-section"の場合の特別な表示処理を追加
   - 言語セクションヘッダーとして表示（カードスタイルではなく、セクション区切りとして）
   - 番号は付けない（セクションヘッダーなので）

2. **スタイリングの実装**
   - 言語セクションヘッダーのデザイン
     - 大きめのフォントサイズ
     - 下線または区切り線
     - 適切なマージン
   - 通常のリポジトリカードとの視覚的な差別化

3. **番号付けロジックの調整**
   - セクションヘッダーは番号付けから除外
   - 各言語セクション内で番号をリセットするか、通し番号にするかを決定
   - 推奨：通し番号（Hacker Newsと同じ）

4. **レスポンシブ対応**
   - モバイル表示でも適切に表示されることを確認

## 期待される成果:
- 言語別セクションが視覚的に区別される
- 各リポジトリは番号付きカードとして表示
- セクションヘッダーは番号なしで表示
- 全体的にHacker Newsと統一感のあるデザイン
