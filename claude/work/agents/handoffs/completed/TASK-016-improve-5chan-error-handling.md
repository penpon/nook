# TASK-016: 5chanエラーハンドリングの改善

## タスク概要
5chanのスレッド取得時に404エラーが発生した場合のエラーハンドリングを改善します。

## 問題の詳細
- スレッド番号1750661869で404エラーが発生
- 複数の代替URLを試しているが、全て失敗
- エラーが発生してもプログラムは継続しているが、ログが冗長

## 変更予定ファイル
- nook/services/fivechan_explorer/（該当ファイルを特定する必要あり）

## 前提タスク
- なし（独立したタスク）

## 実装内容

1. 404エラーの場合は早期にスキップ
2. 代替URLの試行回数を制限（例：3回まで）
3. エラーログの簡潔化

## 実装方針

1. スレッドアクセス時のエラーハンドリングを改善
   - 404エラーは「スレッドが存在しない」として扱い、警告レベルのログに留める
   - 連続した404エラーの場合は代替URLの試行を早期に中止

2. ログメッセージの改善
   - 冗長な「代替URLを試しています」メッセージを削減
   - 最終的な結果のみをログに記録

3. スレッド存在チェックの追加（オプション）
   - スレッドにアクセスする前に、軽量なHEADリクエストで存在確認

## 注意事項
- 5chanの仕様変更に対応できるよう、柔軟な実装にする
- 正常なスレッドへのアクセスには影響を与えない
- エラー統計を取れるようにする（将来の改善のため）
