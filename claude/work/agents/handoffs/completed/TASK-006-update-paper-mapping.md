# TASK-006: PAPER_SUMMARY_TITLE_MAPPINGを実際のデータに合わせて更新

## タスク概要: 論文要約のタイトル変換機能が動作していない問題を修正

## 変更予定ファイル: 
- nook/api/routers/content.py

## 前提タスク: なし

## worktree名: worktrees/TASK-006-update-paper-mapping

## 作業内容:

### 1. 問題の背景
- `PAPER_SUMMARY_TITLE_MAPPING`の質問文キーと実際の論文データの質問文が一致していない
- そのため、`convert_paper_summary_titles`関数による変換が機能していない

### 2. 修正内容
`nook/api/routers/content.py`の`PAPER_SUMMARY_TITLE_MAPPING`を以下のように更新：

```python
PAPER_SUMMARY_TITLE_MAPPING = {
    "1. 既存研究では何ができなかったのか": "🔍 研究背景と課題",
    "2. どのようなアプローチでそれを解決しようとしたか": "💡 提案手法",
    "3. 結果、何が達成できたのか": "🎯 主要な成果",
    "4. Limitationや問題点は何か": "⚠️ 限界と今後の課題",  # 修正
    "5. 技術的な詳細について": "🔧 技術詳細",
    "6. コストや物理的な詳細について": "💻 計算リソースと規模",
    "7. 参考文献のうち、特に参照すべきもの": "📚 重要な関連研究",
    "8. この論文を140字以内で要約すると？": "📝 140字要約"  # 修正
}
```

### 3. 注意事項
- 質問文の後に追加説明が付いている場合があるため（例：「本文で言及されているものの他、あなたが考えるものも含めて」）、基本的な質問文のみをキーとする
- `convert_paper_summary_titles`関数は文字列の完全一致で置換を行うため、キーは正確に記述する必要がある

### 4. テスト方法
1. 修正後、開発サーバーを再起動
2. ブラウザで http://localhost:5173/ にアクセス
3. ArXivボタンをクリック
4. 論文要約の質問部分が絵文字付きのタイトルに変換されていることを確認

### 5. 期待される結果
- 「4. Limitationや問題点は何か」→「⚠️ 限界と今後の課題」
- 「8. この論文を140字以内で要約すると？」→「📝 140字要約」
- 他の質問文も対応する絵文字付きタイトルに変換される
